# Example: Project Product + Architecture Assessment (Template)

## Scenario Signals
- User asks for holistic assessment of current project product capabilities and technical architecture design.
- Assessment requires cross-role perspectives rather than single-role code review only.

## Parameterized Inputs
- Project context: `[project_scope]`
- Assessment focus: `[product_functionality | architecture_design | implementation_risks]`
- Constraints: `[timeline | reliability | cost | maintainability]`

## Suggested Pattern
- Pattern: `Triad Trade-off` (default) or `Cross-functional Panel` for high-complexity projects.
- Candidate roles: `[product] + [architect] + [backend]`
- Optional roles: `[frontend] [algorithm] [domain_expert]` based on uncertainty map.
