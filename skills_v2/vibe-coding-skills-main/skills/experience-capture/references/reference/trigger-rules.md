# Trigger Rules

## 1. 触发策略
- 默认策略：`manual-first`（用户主动触发优先）。
- 可选策略：`suggest-once`（系统建议一次是否保存经验）。

## 2. 决策优先级（必须遵守）
1. 先判断是否为用户显式触发（manual-explicit）。
2. 若显式触发命中，直接进入 skill 流程（不被低打扰/冷却拦截）。
3. 仅在非显式触发时，才进入 `suggest-once` 分支并应用冷却与反触发规则。

## 3. 主动触发条件（任一满足）
- 用户明确提出：经验沉淀 / 复盘 / lessons learned / 保存经验。
- 用户明确提出：总结上面讨论的经验形成记录 / 形成经验记录 / 沉淀经验卡。
- 用户要求将本次方法沉淀到知识库供后续复用。

## 4. 建议触发硬条件（全部满足）
- 本轮存在多轮反复对齐（至少 3 次关键纠偏）。
- 形成明确共识或完成关键修正。
- 可抽象出跨项目适用的规则或清单。
- 最近同主题未触发过建议（满足冷却）。

## 5. 反触发条件（任一满足）
- 仅是一次性小修复。
- 仅有进度同步，没有可迁移经验。
- 用户刚拒绝保存，仍在冷却窗口。
- 仅要求原文归档（verbatim transcript），未要求抽象提炼。

## 6. 交互约束
- 单个候选节点最多询问 1 次。
- 用户拒绝后进入冷却窗口，期间不重复提醒。
- 未得到用户确认，不执行写入。
- 显式触发时，允许跳过“建议询问”步骤，直接进入抽取与写入确认。

## 7. 冷却建议
- topic 级冷却：默认 30 分钟。
- 会话级冷却：同一主题一轮对话内不重复询问。

## 8. 说明
- 低误报通过规则门禁与交互约束实现。
- 本 skill 不引入“评分计算引擎”作为核心机制。
- “低打扰”只约束自动建议触发，不约束用户显式触发。
