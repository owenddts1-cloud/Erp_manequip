# Activation Cues

## Trigger Cues
Activate when at least one cue is present:
- User asks for verification, official sources, or latest information.
- Current claim depends on external and potentially changing facts.
- Multi-round disagreement exists on one technical point with weak evidence.
- Decision depends on rapidly evolving AI model capability comparisons.
- Decision depends on current ecosystem adoption status (for example, MAS landing state).

## Non-Trigger Cues
Do not activate when:
- Local code/tests/logs can provide direct verification.
- Task is deterministic and does not rely on external facts.
- Request is memory capture or experience-card creation.

## Operational Hint
- If trigger confidence is low and user intent is ambiguous, ask one short confirmation question before searching.
- Keep this as cue-based guidance; do not turn it into a weighted scoring system.
