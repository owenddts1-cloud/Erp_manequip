# Source Priority

## Priority Order
1. Official product/framework documentation and release notes.
2. Official repository content (maintainer guides, tagged releases, changelogs).
3. Standards bodies and peer-reviewed papers.
4. Reputable technical publications or community posts (only as supporting evidence).

## Trust Rules
- Prefer first-party sources when available.
- Always capture date/version context when source provides it.
- Use at least two independent sources for high-impact decisions.
- If sources conflict, report the conflict and mark verdict as `inconclusive` or `revised`.
