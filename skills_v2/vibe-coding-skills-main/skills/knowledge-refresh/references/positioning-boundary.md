# Knowledge Refresh: Positioning and Boundary

## Positioning
- Goal: improve coding decision quality when prior knowledge may be stale or insufficient.
- Value: avoid wrong architectural/tooling choices caused by outdated assumptions.
- Nature: a lightweight verification workflow skill, not a runtime system.

## In Scope
- Decide when external verification is needed.
- Prefer authoritative sources and check freshness.
- Return concise verdict with evidence pointers and decision impact.

## Out of Scope
- Running local debugging workflows.
- Persisting project continuity memory.
- Persisting reusable experience cards.
- Building trigger scoring or orchestration engines.

## Relationship with Other Skills
- `layered-project-memory`: project continuity and state/event memory.
- `experience-capture`: reusable lessons and cross-task patterns.
- `knowledge-refresh`: external claim verification for current decision quality.
