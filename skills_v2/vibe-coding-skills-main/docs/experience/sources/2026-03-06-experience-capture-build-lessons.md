# Source Thread: Experience-Capture Skill Build Lessons (2026-03-06)

## skill-value-boundary
- Topic: skill 的价值定位与职责边界。
- Conclusion: skill 提供先验知识与流程引导，不承担项目运行时编排功能。
- Outcome: experience-capture 与 layered-project-memory 形成正交关系（可引用，不重叠）。

## trigger-priority
- Topic: 触发策略与低打扰冲突。
- Conclusion: 用户显式触发优先级最高，不应被低打扰规则拦截。
- Outcome: 低打扰和冷却只约束 suggest-once 自动建议分支。

## quality-and-portability
- Topic: 经验卡质量门禁与可迁移性。
- Conclusion: 经验卡必须可追溯、可检索、可移植，引用优先用相对路径和明确锚点。
- Outcome: 固化 source_event_ref/doc_ref 约束与 capture_meta 审计字段。
